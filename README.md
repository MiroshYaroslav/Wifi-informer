Смарт-дисплей / Інформер (клієнтська частина)

Мета проекту
Розробити стаціонарний настільний смарт-дисплей для відображення актуальної інформації: часу, погоди, фінансових даних (курси валют або криптовалют), а також статусу локального сервера. У майбутньому систему планується розширити для інтеграції з Home Assistant та базового керування елементами розумного дому.

Архітектура системи
Проект побудований за моделлю client-server.

Клієнтська частина (ESP32)
Мікроконтролер відповідає за:

підключення до Wi-Fi;

відображення графічного інтерфейсу за допомогою LVGL;

обробку дотиків через сенсорний екран;

періодичне отримання готових даних із локального сервера.

ESP32 не виконує складних обчислень і не звертається безпосередньо до зовнішніх API. Оновлення даних запускається через інтервал часу за допомогою millis().

Серверна частина (FastAPI, у планах)
Локальний Python-сервер виконує роль проміжного шару:

отримує дані з інтернет-джерел;

взаємодіє з Home Assistant;

обробляє інформацію;

повертає клієнту компактний JSON із уже підготовленими даними.

Такий підхід зменшує навантаження на ESP32, спрощує клієнтський код і робить систему масштабованою.

Апаратна платформа

Плата: CYD (Cheap Yellow Display) ESP32-2432S028 на базі ESP32-WROOM-32

Дисплей: 2.8" TFT SPI, контролер ILI9341, роздільна здатність 320×240, альбомна орієнтація

Сенсор: резистивний тачскрин на контролері XPT2046

Середовище розробки та технології

IDE: CLion

Система збірки: PlatformIO

Фреймворк: Arduino

Графічна бібліотека: LVGL 8.x

Стиль коду: modern C++ з використанням constexpr, nullptr, const correctness, static_cast та акуратної модульної структури

Особливості реалізації

інтерфейс організований у вигляді вкладок;

дані оновлюються через фіксований часовий інтервал;

клієнт відображає лише готову інформацію, отриману від локального сервера;

закладена основа для подальшого розширення функціоналу без повного переписування архітектури.
